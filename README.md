# Quizou 2.0 — Advanced AI Scholar Bowl Platform

**Quizou** is a free, AI-powered Scholar Bowl practice platform designed for the **Congressional App Challenge**. It generates NAQT-style pyramidal questions across 50+ academic subjects, tracks performance with adaptive analytics, and provides innovative study tools — all aimed at making competitive quiz preparation accessible to every student, especially those at underfunded schools.

Built with **Flask**, **SQLite**, and **Together.AI (Meta Llama 3.1)**, Quizou delivers a full-featured learning environment in a lightweight, deployable package.

## Features

### Core Platform
- **AI-Powered NAQT Questions** — Pyramidal toss-up questions (hard clues first, easy giveaway last) generated by Meta Llama 3.1
- **50+ Topics** — Science, History, Literature, Fine Arts, Mathematics, Geography, Social Science, and more
- **3 Difficulty Levels** — Easy, Medium, Hard with adaptive recommendations
- **Global Answer Deduplication** — No user ever sees the same answer twice, across all modes
- **Answer Protection** — AI answers never leak into question text (multi-layer validation)
- **Voice & Smart Answer Checking** — Text-to-Speech, Speech-to-Text, and fuzzy matching (65% Levenshtein threshold)

## Architecture

```
┌──────────────────────────────────────────────────────┐
│                    Frontend (Vanilla JS)              │
│  28 HTML pages, 27 JS files, 8,190 lines of CSS      │
│  Multi-page app with server-side routing              │
└──────────────────────┬───────────────────────────────┘
                       │ RESTful JSON API (109 routes)
┌──────────────────────▼───────────────────────────────┐
│                 Flask Backend (Python)                 │
│  main.py (3,300+ lines) — Routes, auth, sessions      │
│  ai.py (1,190+ lines) — 14 AI generation functions    │
│  migrate_db.py — Idempotent DB migrations             │
└──────────────────────┬───────────────────────────────┘
                       │
          ┌────────────┴────────────┐
          ▼                         ▼
┌──────────────────┐    ┌────────────────────────┐
│   SQLite (30      │    │   Together.AI API       │
│   tables)         │    │   Meta Llama 3.1 8B     │
│   scholar_bowl.db │    │   Instruct Turbo        │
└──────────────────┘    └────────────────────────┘
```

### Key Design Decisions
- **Vanilla JS** — No framework dependencies; fast load times, simple deployment
- **SQLite** — Zero-config embedded database; single-file portability
- **Server-side routing** — Flask serves HTML pages; client-side JS handles interactivity
- **Two-layer dedup** — Prompt-level instructions + post-generation DB filtering ensure unique questions
- **Question completeness validation** — Every generated question is validated for proper NAQT structure (3+ sentences, proper "Name this..." ending, no mid-sentence truncation)

## Getting Started

### Prerequisites
- Python 3.8+
- A [Together.AI](https://together.ai) API key (free tier available)

### Installation

```bash
# Clone the repository
git clone https://github.com/YOUR_USERNAME/quizou.git
cd quizou

# Install dependencies
pip install flask flask-cors requests werkzeug

# Set environment variables
export SESSION_SECRET="your-random-secret-key"
export TOGETHER_API_KEY="your-together-ai-api-key"

# Run database migrations
python migrate_db.py

# Start the application
python main.py
```

The app will be available at `http://localhost:5000`.

### Quick Start
1. Open the app and **Sign Up** (choose Independent, Student, or Teacher role)
2. Select a **topic** (e.g., "American History"), **difficulty**, and **number of questions**
3. Click **Generate Questions** — AI creates unique NAQT-style toss-ups
4. Answer questions, track your accuracy, and explore advanced features from the home page

## Question Generation

### NAQT Pyramidal Format
Every question follows the standard NAQT competition format:

```
[HARD CLUES] → [MEDIUM CLUES] → [EASY CLUES] → [GIVEAWAY] → "Name this [type]."
```

**Example:**
> This 19th-century scientist conducted experiments with pea plants in a monastery garden. 
> His work on heredity was largely ignored during his lifetime but was rediscovered in 1900. 
> He formulated fundamental principles explaining how traits are inherited through discrete units. 
> His laws include the Law of Segregation and the Law of Independent Assortment. 
> He is now considered the father of modern genetics. 
> Name this Austrian monk and scientist.
>
> **Answer: Gregor Mendel**

### Quality Assurance Pipeline
1. **Prompt Engineering** — System prompts enforce pyramidal structure, answer protection, and diversity
2. **Answer Protection** — Multi-layer validation ensures the answer never appears in question text (stop-word normalization, significant word checking, reverse name matching)
3. **Completeness Validation** — Every question checked for 3+ sentences, proper ending ("Name this..."), and no mid-sentence truncation
4. **Global Deduplication** — Two-layer approach (prompt-level + DB-level) prevents duplicate answers per user across all modes
5. **Retry Logic** — Failed generations retry up to 3 times with different prompts

## Advanced Features

### Question Autopsy Lab
Dissects missed questions clue-by-clue. AI identifies the specific knowledge gap, provides a mini-lesson with key facts, and generates targeted recovery quiz questions. Accessible directly from the Missed Questions review page via the "Dissect" button.

### AI Debate Arena
3-round academic debate against AI on any topic. The system randomly assigns sides (for/against). Each round, the student submits an argument and the AI responds with a counter-argument. After all rounds, AI scores the debate on factual accuracy, reasoning quality, and evidence use (1-10 each) with detailed feedback.

### Visual Timeline Builder
Generates historical events for any topic and shuffles them. Students drag-and-drop events into chronological order. After checking, the system reveals correct dates, context for each event, and scores placement accuracy. Supports both mouse drag and touch interfaces.

### Power Training
Reveals NAQT questions one clue at a time. Buzzing early earns more points (15 pts for clues 1-2, 10 pts for 3-4, 5 pts for 5+, -5 pts for wrong buzzes). Tracks career stats including average buzz depth and total power points.

### Knowledge Decay Tracker
Models topic freshness using exponential decay based on days since last practice and historical accuracy. Color-coded health bars (green/yellow/red) show which topics need review, with direct "Review Now" links that pre-fill the topic.
