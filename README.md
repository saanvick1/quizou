**Quizou** is a free, AI-powered Scholar Bowl practice platform that generates unlimited NAQT-style pyramidal questions across 50+ academic subjects, tracks performance with adaptive analytics, and provides innovative study tools — all aimed at making competitive quiz preparation accessible to every student, especially those at underfunded schools.

Built with **Flask**, **SQLite**, and **Together.AI (Meta Llama 3.1)**, Quizou delivers a full-featured learning environment in a lightweight, deployable package.

### Competitor Breakdown

#### Protobowl
> **What it does:** Real-time online Quiz Bowl buzzer practice with a fixed question database.
>
> **Why Quizou is better:** Protobowl's question database is finite — once you've practiced enough, you start recognizing answers from memory rather than knowledge. Quizou generates fresh, unique questions every time using AI, so you're always learning new material. Protobowl also has no analytics, no adaptive difficulty, no study tools, and no AI coaching.

#### QBReader
> **What it does:** Archive of past Quiz Bowl questions for reading practice.
>
> **Why Quizou is better:** QBReader is a static archive — no new questions, no AI explanations, no personalized learning path. Quizou provides unlimited fresh questions plus 37 features including concept mapping, knowledge decay tracking, AI debate, and question autopsy that QBReader simply doesn't have.

#### Kahoot!
> **What it does:** Classroom game-based quizzes with live multiplayer sessions.
>
> **Why Quizou is better:** Kahoot requires teachers to manually create every quiz and uses a multiple-choice format — nothing like real NAQT competition. Quizou auto-generates proper pyramidal questions on any topic instantly. Kahoot has no AI coaching, no individual analytics beyond basic scores, no competition simulation, and costs money for premium features.

#### Quizlet
> **What it does:** User-created flashcard sets with various study modes.
>
> **Why Quizou is better:** Quizlet relies entirely on user-created content — quality is inconsistent and sets must be manually built. Quizou generates high-quality NAQT-format questions instantly on any topic. Quizlet has no pyramidal format, no competition modes, no AI coaching, no knowledge decay tracking, and charges for advanced features.

#### Quizizz
> **What it does:** Self-paced classroom quizzes with gamification.
>
> **Why Quizou is better:** Like Kahoot, Quizizz depends on teacher-created content and uses a multiple-choice format. It has no competition simulation, no power training, no question autopsy, no knowledge decay tracking, and no AI-powered study tools. Quizou provides all of these for free.

#### Brainscape
> **What it does:** Adaptive flashcards with confidence-based repetition.
>
> **Why Quizou is better:** Brainscape is flashcard-only — no quiz bowl format, no AI question generation, no competition modes, no debate arena, no timeline builder. Its adaptive algorithm is limited to card repetition frequency, while Quizou offers full cognitive gap analysis, knowledge decay modeling, and concept mapping.

## Architecture

```
┌──────────────────────────────────────────────────────┐
│                    Frontend (Vanilla JS)              │
│  28 HTML pages, 27 JS files, 8,190+ lines of CSS     │
│  Multi-page app with server-side routing              │
└──────────────────────┬───────────────────────────────┘
                       │ RESTful JSON API (109+ routes)
┌──────────────────────▼───────────────────────────────┐
│                 Flask Backend (Python)                 │
│  main.py (3,300+ lines) — Routes, auth, sessions      │
│  ai.py (1,190+ lines) — 14 AI generation functions    │
│  migrate_db.py — Idempotent DB migrations             │
└──────────────────────┬───────────────────────────────┘
                       │
          ┌────────────┴────────────┐
          ▼                         ▼
┌──────────────────┐    ┌────────────────────────┐
│   SQLite (30      │    │   Together.AI API       │
│   tables)         │    │   Meta Llama 3.1 8B     │
│   scholar_bowl.db │    │   Instruct Turbo        │
└──────────────────┘    └────────────────────────┘
```

### Key Design Decisions
- **Vanilla JS** — No framework dependencies; fast load times, simple deployment
- **SQLite** — Zero-config embedded database; single-file portability
- **Server-side routing** — Flask serves HTML pages; client-side JS handles interactivity
- **Two-layer dedup** — Prompt-level instructions + post-generation DB filtering ensure unique questions
- **Question completeness validation** — Every generated question is validated for proper NAQT structure (3+ sentences, proper ending, no mid-sentence truncation)
- **Auto-rephrasing** — If the AI accidentally includes the answer in the question text, the system automatically rephrases instead of rejecting

## Getting Started

### Prerequisites
- Python 3.8+
- A [Together.AI](https://together.ai) API key (free tier available)

### Installation

```bash
git clone https://github.com/saanvick1/quizou.git
cd quizou

pip install flask flask-cors requests werkzeug

export SESSION_SECRET="your-random-secret-key"
export TOGETHER_API_KEY="your-together-ai-api-key"

python migrate_db.py

python main.py
```

The app will be available at `http://localhost:5000`.

Or try the live version at **[quizou.replit.app](https://quizou.replit.app)**.

### Quick Start
1. Open the app and **Sign Up** (choose Independent, Student, or Teacher role)
2. Select a **topic** (e.g., "American History"), **difficulty**, and **number of questions**
3. Click **Generate Questions** — AI creates unique NAQT-style toss-ups
4. Answer questions, track your accuracy, and explore all 37 features from the home page

## Question Generation

### NAQT Pyramidal Format
Every question follows the standard NAQT competition format:

```
[HARD CLUES] → [MEDIUM CLUES] → [EASY CLUES] → [GIVEAWAY] → "Name this [type]."
```

**Example:**
> This 19th-century scientist conducted experiments with pea plants in a monastery garden. 
> His work on heredity was largely ignored during his lifetime but was rediscovered in 1900. 
> He formulated fundamental principles explaining how traits are inherited through discrete units. 
> His laws include the Law of Segregation and the Law of Independent Assortment. 
> He is now considered the father of modern genetics. 
> Name this Austrian monk and scientist.
>
> **Answer: Gregor Mendel**

### Quality Assurance Pipeline
1. **Prompt Engineering** — System prompts enforce pyramidal structure, answer protection, and diversity
2. **Answer Protection** — Multi-layer validation ensures the answer never appears in question text (stop-word normalization, significant word checking, reverse name matching)
3. **Auto-Rephrasing** — If an answer is detected in the question text, the system automatically calls the AI to rephrase instead of rejecting
4. **Completeness Validation** — Every question checked for 3+ sentences, proper ending, and no mid-sentence truncation
5. **Global Deduplication** — Two-layer approach (prompt-level + DB-level) prevents duplicate answers per user across all modes
6. **Retry Logic** — Failed generations retry up to 3 times with different prompts
7. **JSON Repair** — Malformed AI responses are automatically repaired with regex extraction and trailing-comma cleanup

## Advanced Features

### Question Autopsy Lab
Dissects missed questions clue-by-clue. AI identifies the specific knowledge gap, provides a mini-lesson with key facts, and generates targeted recovery quiz questions. Accessible directly from the Missed Questions review page via the "Dissect" button.

### AI Debate Arena
3-round academic debate against AI on any topic. The system randomly assigns sides (for/against). Each round, the student submits an argument and the AI responds with a counter-argument. After all rounds, AI scores the debate on factual accuracy, reasoning quality, and evidence use (1-10 each) with detailed feedback.

### Visual Timeline Builder
Generates historical events for any topic and shuffles them. Students drag-and-drop events into chronological order. After checking, the system reveals correct dates, context for each event, and scores placement accuracy. Supports both mouse drag and touch interfaces.

### Power Training
Reveals NAQT questions one clue at a time. Buzzing early earns more points (15 pts for clues 1-2, 10 pts for 3-4, 5 pts for 5+, -5 pts for wrong buzzes). Tracks career stats including average buzz depth and total power points.

### Knowledge Decay Tracker
Models topic freshness using exponential decay based on days since last practice and historical accuracy. Color-coded health bars (green/yellow/red) show which topics need review, with direct "Review Now" links that pre-fill the topic on the quiz page.

### Interactive Concept Map
Auto-builds a force-directed knowledge graph from your practice history. Hexagonal nodes represent topics, sized by practice volume, with colored mastery rings (green = mastered, orange = learning, red = beginner). Features drag interaction, zoom, particle effects, and click-to-view detail panels.

### AI Study Coach
ChatGPT-style conversational interface that analyzes your performance data, behavior patterns, concept mastery, and cognitive gaps to provide personalized study plans and advice.

## Accessibility

Quizou is designed with WCAG compliance in mind:
- Skip navigation links
- ARIA landmarks (`role="banner"`, `role="navigation"`, `role="main"`)
- `aria-labels` on all inputs and buttons
- `aria-live` regions for dynamic content updates
- `focus-visible` outlines for keyboard navigation
- Keyboard-navigable flashcards (Space/Enter to flip, Arrow keys to navigate)

## Database Schema

The SQLite database contains 30 tables organized by feature area:

**Core:** `users`, `questions`, `history`, `explanations`  
**Analytics:** `user_analytics`, `badges`, `leaderboard`  
**Classroom:** `classrooms`, `classroom_members`, `assignments`, `class_posts`  
**AI Memory Graph:** `concept_nodes`, `concept_edges`, `concept_mastery`, `user_behavior`, `learning_patterns`, `competition_predictions`  
**Learning Tools:** `daily_challenges`, `daily_challenge_results`, `study_sets`, `study_set_questions`  
**Advanced Features:** `knowledge_decay`, `h2h_challenges`, `question_writing`, `power_training_sessions`  
**Innovative Features:** `debate_sessions`, `timeline_attempts`  

All tables use idempotent migrations managed by `migrate_db.py`.
